title: "Qu'est-ce que du bon code™ ?"
---

ESLint, PSR, gofmt, PEP 8, il existe tout un tas de normes de codes pensées pour écrire du code "propre". Et malgré tout ça, tout le monde a déjà croisé du code "moche", même dans des code bases récentes et bien surveillées. Qu'est-ce qui fait du code sale ? Qu'est-ce qui fait du code propre ? D'ailleurs, est-ce qu'un bon développeur écrit toujours du code propre ? Est-ce qu'un code sale est toujours indicateur d'un mauvais développeur ?
Venez pour partir avec beaucoup de questions, peu de réponses, mais peut être quelques pistes de réflexion pour mieux appréhender le problème !
